How the Model Works

1. URL or metrics entered → 2. Public metrics checked (URL mode) → 3. Form filled → 4. API receives JSON → 5. ML model predicts → 6. Risk score returned

The dashboard sends metrics to POST /predict. The trained Random Forest returns outage_predicted and outage_probability, then explanation cards highlight risky inputs.

How to Predict Any Website

  1. Type a public website URL and click Check Website Metrics.
  2. The app measures status code, response time, latency, and simple error rate from 5 probes.
  3. Those values auto-fill the prediction form.
  4. CPU and memory are private — use real values if you own the website, or edit the demo defaults.
  5. Click Predict Outage Risk.
  6. Read outage probability and the risk explanation cards.

What Public URL Checking Can and Cannot Do

Can measure:
  • HTTP status code
  • Response time
  • Basic latency (P95 from probes)
  • Whether the website responds
  • Simple error rate from repeated checks
Cannot measure unless you own the website:
  • Real CPU usage
  • Real memory usage
  • Internal request count
  • Server logs
  • Full application error rate
  • Infrastructure health

Real-World Workflow

  1. Azure Monitor or Application Insights collects live website metrics.
  2. Metrics are sent to the prediction API.
  3. The model predicts outage probability.
  4. If risk is high, the DevOps team gets alerted.
  5. Engineers investigate before the website goes down.
